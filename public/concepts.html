CH 01

계통 구조 — 이 계통이 어떤 계통인가System Topology — Bus-Branch Model & Voltage Levels

Bus-Branch 모델이란?

전력계통 해석에서는 복잡한 실제 설비를 버스(Bus, 모선)와 브랜치(Branch, 선로/변압기) 두 요소로 단순화합니다. 버스는 전기적으로 같은 전압 레벨에서 연결된 점, 브랜치는 두 버스를 잇는 임피던스 요소입니다. 이 추상화 덕분에 계통 전체를 행렬(Ybus)로 표현하고 컴퓨터로 풀 수 있습니다.

핵심 규칙:
① 버스 = 전압이 동일한 전기적 접속점 (발전기 단자, 변전소 모선, 부하 접속점)
② 브랜치 = 두 버스 사이의 임피던스 (선로 r+jx, 변압기 누설 임피던스)
③ 계통 전체 = 버스 집합 + 브랜치 집합 → Ybus 행렬로 표현

한국 전압체계와 이 시뮬레이터의 대응

실제 한국 전력계통은 여러 전압 레벨을 거쳐 전력을 전송합니다:

발전기 출력단 ~20 kV (발전기 단자전압) | [승압변압기 step-up] 20kV --> 345kV 송전망 154 kV / 345 kV / 765 kV (장거리 고압 송전) | [강압변압기 step-down] 345kV --> 154kV 1차 변전망 154 kV (지역 변전소) | [강압변압기] 154kV --> 22.9kV 배전망 22.9 kV → 수용가 220 / 380 V
이 시뮬레이터의 단순화: 실제 계통에는 발전기 단자(~20kV)에서 송전망(345kV)으로 올리는 승압변압기(step-up TR)가 반드시 있습니다. 그러나 이 시뮬레이터는 교육 목적상 이 단계를 생략하고, Bus1(Slack)과 Bus2(PV)를 이미 345kV 송전 레벨의 발전기 모선으로 가정합니다. 변압기 TR을 거쳐 154kV로 강압되어 부하(Bus3·4·5)에 공급됩니다. 이렇게 두면 90MW 부하가 154kV에서 약 340A로, 실제 154kV 계통 전류와 유사해집니다.

CH 02

퍼유닛(per-unit) 시스템Per-Unit System

전압등급이 다른 구간(이 계통은 345kV·154kV)을 실제 단위로 계산하면 변압기 권선비가 식을 복잡하게 만듭니다. pu는 각 구간 정격으로 무차원화해 변압기를 "투명"하게 만듭니다.

전압 pu (구간마다 기준 다름)

V[pu] = V[kV] / V_base[kV] // Bus1,2,6: base 345kV // Bus7,3,4,5: base 154kV // 1.05pu --> 154x1.05 = 161.7kV

전력 pu (계통 전체 하나)

P[pu] = P[MW] / 100MVA // S_base = 100MVA 고정 // 90MW --> 0.90pu // 변압기 있어도 동일 기준
핵심: 전압 pu의 기준(V_base)은 변압기마다 바뀌지만, 전력 pu의 기준(100MVA)은 계통 전체에서 하나로 고정.
임피던스 환산: Z[pu] = Z[Ohm] x S_base / V_base^2 코드의 r,x는 이미 pu값으로 입력되어 별도 환산 불필요.
CH 03

임피던스·어드미턴스의 출처Where Do r, x, y Come From?

r + jx 값은 어떻게 정했나?

이 값들은 실측이 아니라 전압등급·선로특성을 현실적으로 흉내 낸 표준 pu 예시값입니다. 핵심 지표는 X/R 비율로, 송전계통은 항상 X >> R (도체가 굵어 저항은 작고 인덕턴스가 지배적)입니다.

선로r (pu)x (pu)X/R특성
선로1,2 (345kV 송전)0.00150.01510굵은 송전선, 저저항
변압기 TR (345/154kV)0.0010.0440누설리액턴스 큼 → X/R 최대
선로4,5,6 (154kV 부하선)0.020.105가늘고 긴 2차 선로 → 임피던스 큼

어드미턴스 = r+jx 통째로의 복소수 역수

y = 1/(r+jx) = (r - jx)/(r^2+x^2) = g + jb 선로1: 1/(0.0015+j0.015) = (0.0015-j0.015)/0.00022725 = 6.60 - j66.01 TR : 1/(0.001+j0.04) = (0.001-j0.04)/0.0016010 = 0.62 - j24.98 선로4: 1/(0.02+j0.10) = (0.02-j0.10)/0.0104 = 1.92 - j9.62

네, "통째로를 각각 역수"가 맞습니다. 단 복소수 역수이므로 분모를 유리화(r²+x²로 나눔)합니다.

Ybus 행렬 — KCL 유도와 부호 규칙Bus Admittance Matrix

대각항 = "연결된 모든 선로 어드미턴스의 합"

Y11 = 6.60-j66.01 (Bus1엔 선로1 하나만) Y66 = (6.60-j66.01)+(6.60-j66.01)+(0.62-j24.98) = 13.83 - j157.00 (선로1+2+TR) Y77 = (0.62-j24.98)+3x(1.92-j9.62) = 6.39 - j53.83 (TR+선로4+5+6)

비대각항 Y₁₆ = −6.60+j66.01 은 왜 "부호 반대"인가?

정확한 규칙은 "두 모선 사이 선로 어드미턴스의 부호 반전"입니다:

Y16 = -(1-6 사이 선로 어드미턴스) = -(6.60-j66.01) = -6.60 + j66.01
오해 주의: Y16이 마치 −Y11처럼 보이는 건 우연입니다. 모선1은 선로가 딱 하나라 Y11 = 그 선로값이 되어 그렇게 보일 뿐. 모선6은 다릅니다 — Y66 = 13.83−j157.00 이지만 Y16 = −6.60+j66.01 ≠ −Y66. 항상 선로 어드미턴스 기준입니다.

왜 부호를 반대로? — KCL(전류법칙) 유도

모선 i 주입전류: I_i = Sum_j y_ij*(V_i - V_j) (이웃 j로 흐르는 전류 합) = (Sum_j y_ij)*V_i - Sum_j y_ij*V_j V_i 계수 = +합 (대각 Y_ii) V_j 계수 = -y_ij (비대각 Y_ij)

전류가 전압 차(V_i − V_j)에 비례하기 때문에 내 전압엔 (+), 이웃 전압엔 (−)가 붙습니다. "이웃 전압이 높으면 전류가 나에게 역류한다"의 수학적 표현입니다.

0 = 직접 연결 안 됨

Y13 = 0 → 모선1·3은 직접 선로 없음 (6,7을 거쳐 연결). 0의 위치만 봐도 결선도를 복원할 수 있습니다.

CH 06

Slack 버스와 PV 버스Slack Bus, PV Bus & AVR

왜 Slack 버스가 필요한가?

조류 방정식은 위상각의 차이만 관계하므로, 절대 위상을 하나 고정해야 합니다. 이 기준 모선이 Slack 버스입니다. 동시에 계통의 발전·부하·손실 불일치를 자동으로 메우는 수급 균형자 역할도 합니다.

Slack 버스의 두 역할
① 위상 기준: θslack = 0° 고정 (모든 각도는 이 기준과의 차)
② 균형자: 미지의 총 발전 P, 무효 Q를 자동 공급·흡수 (음수면 흡수)

Slack을 "왜, 어떻게" 정하나

전력조류는 미지수와 방정식의 수가 딱 맞아야 풀립니다. 만약 모든 발전기의 P를 다 지정해버리면 손실(I²R)을 미리 알 수 없다는 모순이 생깁니다. 손실은 조류를 풀어야 나오는데, 발전량은 손실까지 포함해 정해야 하니까요. 이 "닭-달걀" 문제를 푸는 장치가 Slack입니다.

전체 발전 = 전체 부하 + 전체 손실 ↑ 풀기 전엔 모르는 값 → 발전기 하나(Slack)의 P를 비워두고, 나머지를 다 푼 뒤 "모자라거나 남는 만큼"을 그 발전기가 자동으로 메우게 한다 = Slack이 손실 + 불균형을 마지막에 흡수·공급
왜 Bus1을 Slack으로 골랐나: Slack은 계통의 모든 불균형을 순간적으로 받아내야 하므로 가장 크고, 예비력(여유 출력)이 많고, 응답이 빠른 주력 발전기가 맡습니다. 실제 계통에서는 대형 기력·원자력·수력 같은 기저 발전기나 계통연계점(타 변전소와의 연결)이 이 역할을 합니다.

PV 버스와 AVR

PV 버스는 유효출력 P와 전압 V가 고정된 발전기 모선. 무효 Q와 위상 θ는 조류해석이 자동 결정합니다.

Slack (Bus1)PV (Bus2)
내가 지정V, θ(=0)P, V
해석이 계산P, Qθ, Q
P 출력자동 (불균형·손실 흡수)내가 고정 (예: 50MW)
역할균형자 + 위상기준지정량 발전 + 전압유지
왜 PV는 P를 고정할 수 있나? 실제 발전기는 운전원이 "이 발전기 60MW로 돌려"라고 출력 지령(거버너 설정)을 내립니다. 단 한 대(Slack)만은 손실까지 메우느라 P를 비워두는 것이고, 나머지 발전기는 전부 PV로 P를 지정합니다. 그래서 발전기 N대 = Slack 1대 + PV (N−1)대가 일반적인 구성입니다.

"왜 Q는 고정 안 하나?" 발전기에는 AVR(자동전압조정기)이 달려 있어서, 전압을 1.05pu로 유지하기 위해 무효전력 Q를 실시간으로 자동 조절합니다. 즉 Q는 운전원이 정하는 게 아니라 "전압을 맞추다 보니 결정되는" 종속 변수입니다. 이 계통에서 Bus2는 V=1.05pu를 지키느라 Q=+15.0Mvar을 자동 공급하는 것으로 풀립니다.

Bus2 지정값

  • P = 50 MW
  • V = 1.05 pu

Bus2 해석이 결정

  • θ = +0.073°
  • Q = +15.0 Mvar

AVR이 Q를 조절하는 원리

V_단자 < V_set → 계자전류 I_f 증가 → E_q 증가 → Q 공급 증가 → V 상승 V_단자 > V_set → 계자전류 I_f 감소 → E_q 감소 → Q 흡수 증가 → V 하강 E_q: 발전기 내부 기전력 (계자전류에 비례)
비유: AVR은 자동 수도꼭지. 전압(수압)이 낮아지면 꼭지(계자전류)를 열어 무효전력(물)을 더 보내 압력을 회복합니다. PV 모선에서 V가 고정이라는 건 "이 꼭지가 항상 작동 중"임을 가정.

실제 발전기는 Q 한계(권선 가열)가 있어 Qmax 초과 시 PV→PQ로 전환되어 전압이 떨어집니다. 이 시뮬레이터는 한계 없는 이상 모델입니다.

CH 08

무효전력과 전압의 관계Reactive Power & Voltage

유효전력 P (실일)

  • 실제 일 (회전·열)
  • 단위 MW
  • 위상각 차로 전송
  • 장거리 가능

무효전력 Q (허일)

  • 에너지 왕복 (L↔C)
  • 단위 Mvar
  • 전압 크기 결정
  • 장거리 어려움

왜 Q가 전압의 주역인가?

ΔV ≈ (P*R + Q*X) / V X >> R 이므로: ΔV ≈ Q*X / V (Q가 지배!) P를 2배 → ΔV 거의 불변, Q를 2배 → ΔV 2배 (전압 급락)
물 배관 비유: 유효전력 P = 유량(흐르는 물의 양), 무효전력 Q = 수압. 수압(Q)이 낮으면 아무리 많은 물(P)도 끝단 도달 전 압력이 떨어짐 → Q 부족 = 전압 붕괴 위험.

  • SVC/STATCOM: 전력전자로 Q 실시간 가변
  • 발전기 Q 증대: 계자전류↑ → V 회복
  • 역률 개선: 0.95→0.99 시 Q 소비 약 60% 감소
전압 붕괴(Voltage Collapse): Q 공급이 수요를 못 따라가면 전압이 연쇄 하강. 1996 미 서부, 2003 이탈리아 대정전의 주원인이 Q 부족 전압붕괴였습니다.
CH 09

조류계산(Power Flow) — Newton-RaphsonPower Flow — Newton-Raphson Iteration

1) NR이 하는 일 vs N-1의 관계

NR 자체는 "지금 이 구성에서 전압·조류가 얼마냐"를 한 번 푸는 계산기입니다. "발전기2 탈락 → Bus1이 보충, 선로 부하율 얼마" 같은 시나리오는 N-1 해석이 NR을 조건만 바꿔 반복 호출하는 것입니다.

정상 상태 → NR 1회 실행 → 전압·조류·손실 (현재 부하율) 발전기2 제거 후 → NR 다시 실행 → 새 전압·조류 (Slack이 부족분 자동 보충) 선로3 개방 후 → NR 다시 실행 → 또 다른 전압·조류 ... N-1 = 위 과정을 설비 수만큼 자동 반복
핵심 구분: NR은 한 장면을 정확히 푸는 도구. N-1은 NR을 여러 시나리오마다 호출하는 상위 루프. 시뮬레이터에서 차단기를 토글할 때마다 NR이 새로 돌아가는 게 바로 이 구조입니다.

2) Ybus로 주입전력 계산 (busInjection 공식)

Ybus 자체가 그 공식의 재료입니다. network.ts의 busInjection 함수가 정확히 이 일을 합니다:

P_i = V_i x Sum_j Vj (G_ij*cos(θ_ij) + B_ij*sin(θ_ij)) Q_i = V_i x Sum_j Vj (G_ij*sin(θ_ij) - B_ij*cos(θ_ij)) G_ij, B_ij = Ybus의 실수부·허수부 ← r, x의 역수로 만든 값 θ_ij = θ_i - θ_j ← 두 모선의 위상각 차이

즉 흐름은: Ybus(고정) + 현재 V·θ(반복마다 갱신) → 위 공식 대입 → 각 모선 P, Q 계산. V·θ만 알면 주입전력이 바로 나옵니다. Ybus는 계통 구성이 안 바뀌는 한 매 반복에서 동일하게 재사용됩니다.

3) Flat Start

아무것도 모르는 상태에서 시작. 모든 전압을 1.0pu, 모든 위상각을 0°으로 설정합니다. 단, 발전기(Bus1·2)는 설정값(1.05pu)으로 시작합니다.

초기값: V1=1.05, V2=1.05 ← 발전기는 설정값 V6=V7=V3=V4=V5=1.00 ← 나머지는 1.0으로 가정 θ1=θ2=θ6=θ7=θ3=θ4=θ5 = 0° ← 모든 위상각 0으로 가정

4) mismatch란? (b=66 유도 포함)

NR은 "우리가 원하는 전력(P_spec, Q_spec)"과 "현재 추정한 전압으로 계산한 전력(P_calc, Q_calc)"의 차이를 줄여나가는 과정입니다.

mismatch = P_spec - P_calc(V, θ) Q_spec - Q_calc(V, θ) 이 값이 0에 가까울수록 → 전압 추정값이 실제에 가깝다 = 수렴
직관적 비유: 내가 "버스6에 전력이 0MW 들어오길 원한다"고 지정했는데, 현재 전압 추정값으로 계산해보니 −300MW가 흐른다고 나오면 mismatch = 300MW. 이 차이를 없애도록 전압을 조정하는 것이 NR입니다.

위상각이 전부 0이면 cos(θi−θj)=1, sin(θi−θj)=0이 되어 Q 계산식이 단순해집니다:

Q_calc_i = -Vi x Sum_j(Vj x B_ij) (모든 θ=0일 때 sin=0이라 G항 사라짐) Bus6에 대해 계산: B61=+66.01, B62=+66.01, B66=-157.00, B67=+24.98 (Ybus B행렬에서) Sum = V1*66.01 + V2*66.01 + V6*(-157.00) + V7*24.98 = 1.05*66.01 + 1.05*66.01 + 1.00*(-157.00) + 1.00*24.98 = 69.31 + 69.31 - 157.00 + 24.98 = 6.60 Q_calc_6 = -1.00 x 6.60 = -6.60 pu Q_spec_6 = 0 (Bus6은 부하·발전 없는 집전모선 → Q = 0 이어야 함) ΔQ6 = Q_spec - Q_calc = 0 - (-6.60) = +6.60 pu ← max mismatch!
왜 Bus6의 ΔQ가 가장 클까? — 그리고 B=66은 어디서 나오나?
선로 1·2는 r=0.0015, x=0.015로 정의되어 있습니다. 서셉턴스는 다음과 같이 계산됩니다:
b = -x / (r^2+x^2) = -0.015 / (0.00000225+0.000225) = -0.015 / 0.00022725 = -66.01 Ybus 비대각 원소 B61 = -b = +66.01 (선로 어드미턴스의 부호 반전)
Bus6은 발전기(1.05pu)와 나머지 계통(1.0pu) 사이의 집전모선입니다. 전압 단차(1.05−1.00=0.05pu)와 B=66이 곱해지면:
ΔQ6 ≈ 전압단차 x B x 발전기수 = 0.05 x 66 x 2 = 6.60 pu
선로 임피던스가 작을수록(=B가 클수록) 이 초기 불일치가 커집니다.

5) 야코비안

mismatch를 없애려면 V와 θ를 얼마나 바꿔야 하는지 계산합니다. 이때 사용하는 것이 야코비안(J)입니다.

J x Δx = mismatch J = [∂P/∂θ ∂P/∂V] ← 전압/각도를 조금 바꿀 때 P가 얼마나 변하는지 [∂Q/∂θ ∂Q/∂V] ← 전압/각도를 조금 바꿀 때 Q가 얼마나 변하는지 이 식을 Gauss 소거법으로 풀어 Δθ, ΔV를 구함 → 전압/각도 갱신: θ ← θ + Δθ, V ← V + ΔV

왜 모든 반복에서 "ΔQ at Bus6"가 최대인가?
maxMismatch는 매 반복마다 "지금 가장 안 맞는 곳"을 가리킵니다.
  • iter0·1: flat start 직후엔 발전기(1.05pu)와 집전모선(1.0pu)의 전압 단차 때문에 Bus6의 무효전력(Q)이 가장 안 맞음 → ΔQ6가 최대 (6.601 → 0.363).
  • iter2 이후: 이번 계통은 선로 서셉턴스가 매우 큽니다(b=66). 서셉턴스가 클수록 무효전력이 전압 오차에 민감하게 반응하므로, 위상각·전압이 거의 자리잡은 뒤에도 Bus6의 Q가 가장 빡빡한 제약으로 남습니다. 그래서 수렴이 끝날 때까지 최대 mismatch는 계속 ΔQ@Bus6입니다 (더 이상 ΔP@Bus7로 바뀌지 않음).

7) 이차수렴

뉴턴법은 오차가 매 반복마다 제곱으로 줄어드는 특성이 있습니다. 이를 이차수렴(quadratic convergence)이라 합니다.

오차가 e이면, 다음 반복 후 오차 ≈ k x e^2 iter0 → iter1: 6.601 → 0.363 (18배 감소) iter1 → iter2: 0.363 → 0.00107 ≈ 0.363^2 x 0.008 (339배) iter2 → iter3: 0.00107 → 1.35e-8 ≈ 0.00107^2 (79000배) iter3 → iter4: 1.35e-8 → 4.8e-14 ≈ (1.35e-8)^2 (제곱 그대로!) 유효 자릿수: 1자리 → 1자리 → 3자리 → 8자리 → 13자리 ↑ 2배씩 늘어남 = 이차수렴
이차수렴이 실용적인 이유: 일반 반복법(선형수렴)은 100번 반복해도 2자리 개선이지만, NR은 4번으로 13자리 개선됩니다. 대형 계통(1000개 버스 이상)도 보통 5~10회 안에 수렴하는 이유입니다.

9) 임피던스는 고정인데 부하를 바꿔도 되나 + PF vs 암산

아니요. 임피던스(R, X)는 바뀌지 않습니다. 계통 방정식은 두 종류의 양으로 나뉩니다.

구분무엇무엇이 정하나부하·발전 바꾸면?
네트워크 (Ybus)선로 R, X → G, B전선의 굵기·길이·재질 (물리적 설비)안 바뀜 (고정)
주입 (Injection)P_spec, Q_spec발전기 출력·부하 소비 (운전 조건)바뀜 (당신이 조절)
핵심 직관: R·X는 "수도관의 굵기"입니다. 물(전력)을 많이 흘리든 적게 흘리든 관 자체는 그대로. 부하·PV를 바꾸는 것은 "양 끝에서 물을 얼마나 넣고 빼느냐"를 바꾸는 것일 뿐입니다.

왜 암산이 안 되냐면, 전력조류는 비선형 연립방정식이기 때문입니다.

P, Q = f(V, θ) ← V와 θ가 sin·cos로 얽힌 비선형식 선로 손실 I^2*R 은 전류의 제곱 → 부하 2배면 손실 4배 예) "부하 90MW니까 발전 90MW" (암산) 실제 PF: 발전 = 90MW(부하) + 0.74MW(손실) = 90.74MW 게다가 Bus5 전압이 1.014pu까지 떨어짐 ← 암산으로는 못 봄
한 줄 요약: 암산은 "흐르냐/안 흐르냐"(정성적)를 보고, PF는 "얼마나, 몇 볼트로, 손실은, 위반은"(정량적)을 봅니다. 765kV 수백 모선 계통에서 0.05pu 전압강하나 95% 선로 부하율 같은 위험은 오직 PF로만 잡아냅니다.
CH 10

N-1 해석 — 선로 개방 시나리오N-1 Contingency Analysis

계통해석이란 곧 "개방된 상태의 Ybus를 다시 짜서 조류해석을 푸는 것"입니다. 세 대표 케이스를 실제 엔진으로 확인했습니다.

선로1을 열면 전 계통 정전인데, 선로2를 열면 왜 정전이 아닌가?

핵심은 연결성 탐색(BFS)이 항상 Slack(Bus1)에서 출발한다는 점입니다.

선로1 개방 (Bus1↔Bus6)

Bus1(Slack) -X- Bus6 -- Bus7 -- 부하 Bus2(PV) ──────┘

Bus1(Slack)에서 BFS 시작 → 선로1이 유일한 통로인데 끊김 → 도달 가능: {Bus1}만 → Bus2,6,7,3,4,5 전부 고립 → 90MW 전 계통 정전

선로2 개방 (Bus2↔Bus6)

Bus1(Slack) ──── Bus6 -- Bus7 -- 부하 Bus2(PV) -X-────┘

Bus1(Slack)에서 BFS 시작 → 선로1은 살아있음 → Bus6·7·3·4·5 도달 → Bus2만 고립 (발전기뿐, 부하 없음) → 부하 정전 0MW

왜 Bus2 고립은 괜찮은가?
Bus2는 발전기 전용 모선으로 소비 부하가 없습니다. Bus2가 탈락하면 50MW 발전이 사라지지만, Slack(Bus1)이 균형자 역할로 그 50MW를 자동 보충합니다. 수용가 입장에서는 전력 공급이 끊기지 않습니다.

765-345-154-22.9 계통에서 부하 탈락 시

CH 13

EMS 파이프라인 — Topology·PF·N-1·SE 관계How PF, N-1, SE & Topology Fit Together

이 시뮬레이터에서 하는 게 PF입니까?

네, 맞습니다. 슬라이더로 PV 발전량·부하량을 바꾸거나 차단기를 열고 닫을 때마다 내부에서 Newton-Raphson이 돌아가고, 화면에 나오는 전압·선로 부하율·과부하 경보가 모두 Power Flow(조류계산) 결과물입니다.

입력: 발전량(P, V 지정) + 부하량(P, Q 지정) + 계통 구성(어떤 차단기가 닫혀있나) ↓ Newton-Raphson 출력: 각 모선의 전압·위상각 + 각 선로의 조류·손실·부하율

PF · N-1 · 상태추정은 별개인가, 연결되나?

도구로서는 별개이지만, 실제 EMS에서는 순서대로 쌓아서 씁니다. 아래가 실제 EMS의 실시간 운영 흐름입니다.

① Topology Processing → "지금 어떤 모선이 어떻게 연결돼 있나" 파악 ↓ ② 상태추정 (WLS SE) → "현장 계측값으로 현재 V·θ 확정" ↓ ③ PF → "확정된 상태에서 조류 계산" ↓ ④ N-1 해석 → "선로 하나씩 끊어가며 PF 반복"
개념한 줄 정의방향이 시뮬레이터에서
Topology Processing 모선·선로 배치 + 차단기 상태로 현재 전기적 연결 구조 파악 설비 정보 → Ybus 차단기 ON/OFF 시 BFS로 연결 재파악 → Ybus 재구성
PF 운전 조건을 주면 전압·조류를 예측 조건 → 결과 (순방향) 슬라이더·차단기 조작 시 자동 실행
N-1 해석 설비 하나씩 끊어가며 PF를 반복 PF × 선로 수 차단기 하나씩 수동으로 열어보는 것이 수동 N-1
상태추정 잡음 섞인 계측값으로 현재 상태를 역추정 계측 → 상태 (역방향) 별도 탭, PF와 독립 실행

상태추정 — PF보다 먼저 실행하는 이유

PF는 "발전·부하 조건을 주면 전압·조류를 계산"합니다. 그런데 실제 EMS에서는 발전·부하 조건 자체를 현장 계측기로 읽어옵니다. 그 계측값에는 잡음·오차·고장 미터가 섞여 있어서, 이걸 그대로 PF에 넣으면 틀린 결과가 나옵니다. 그래서 PF 이전에 WLS 상태추정으로 계측값을 통계적으로 정제해서 신뢰할 수 있는 V·θ 스냅샷을 만든 다음 PF를 돌립니다.

상태추정 없이: 잡음 섞인 계측값 → PF → 부정확한 조류 결과 상태추정 있을 때: 잡음 섞인 계측값 → WLS → 정제된 V·θ → PF → 신뢰할 수 있는 결과
한 줄 요약: Topology가 지도를 그리고 → 상태추정이 현재 위치를 확인하고 → PF가 경로를 계산하고 → N-1이 "각 도로가 막히면?" 을 시뮬레이션합니다. 네 가지는 순서가 있는 한 파이프라인이며, 이 시뮬레이터는 그 파이프라인 전체를 하나의 화면에서 교육용으로 단순화해 보여줍니다.
CH 14

WLS 상태추정 — 수식 상세WLS State Estimation

SCADA 계측에는 노이즈가 섞이고, 계측기 고장 시 불량데이터(Bad Data)가 들어옵니다. 상태추정은 여분의 계측을 종합해 가장 그럴듯한 상태(V, θ)를 복원합니다. 아래에서 "왜 이렇게 푸는지"부터 한 단계씩 쌓아갑니다.

STEP 0 — 가장 작은 비유부터 (체중 3번 측정)

상태추정의 핵심 아이디어는 중학교 수준의 "평균"입니다. 체중계로 내 몸무게를 3번 쟀더니 70.1, 69.8, 70.3kg이 나왔다고 합시다. 진짜 몸무게(=상태)는 모르지만, 측정값(=계측)이 여러 개라 평균 70.07kg으로 추정할 수 있습니다.

미지수: 진짜 몸무게 x (1개) 계측: z1=70.1, z2=69.8, z3=70.3 (3개) ← 미지수보다 많음(여분=2) 추정값 xhat = 측정값들과의 오차 제곱합을 최소로 만드는 값 = min Sum(zi - x)^2 --> 미분=0 --> xhat = 평균 = 70.07

여기에 두 가지만 더 붙이면 전력계통 상태추정이 됩니다. ① 계측기마다 정밀도가 다르니 가중치(W)를 준다. ② 계측이 상태와 비선형으로 얽혀 있으니 반복(Gauss-Newton)으로 푼다.

STEP 1 — 무엇을 알고(z), 무엇을 모르나(x)

모르는 것 = 상태벡터 x (13개) θ2...θ7 : 위상각 6개 (θ1=Slack=0° 기준이라 제외) V1...V7 : 전압크기 7개 → 이 13개만 알면 계통의 모든 것을 계산할 수 있다 아는 것 = 계측벡터 z (31개) V 7개 : 각 모선 전압계 sigma_V = 0.004 pu (정밀) Pinj 6개 : 모선 유효주입 (slack 제외) sigma_P = 0.008 pu Qinj 6개 : 모선 무효주입 sigma_P = 0.008 pu Pflow 6개: 선로 유효조류 sigma_P = 0.008 pu Qflow 6개: 선로 무효조류 sigma_P = 0.008 pu 여분(redundancy) = 31 - 13 = 18 ← 이 18개의 잉여가 오차를 걸러낸다
핵심 직관: 미지수가 13개인데 측정을 31개나 합니다. 13개면 딱 풀리는데 왜 31개? 바로 그 18개의 여분 때문에 "노이즈는 평균으로 상쇄"되고 "튀는 값 하나는 도드라져서" 잡힙니다. 여분이 없으면(31=13) 불량을 절대 못 잡습니다.

STEP 2 — 계측함수 h(x): 상태로 계측을 "예측"하기

상태 x를 알면 각 계측이 얼마여야 하는지 계산할 수 있습니다. 이 계산식이 h(x)입니다.

h(x) = "상태 x일 때 이 계측기가 가리켜야 할 값" 전압계: hi(x) = Vi (그냥 그 모선 전압) 선로 유효조류: hi(x) = g*Vi^2 - Vi*Vj*(g*cos(θij) + b*sin(θij)) 주입전력: hi(x) = Sum_j Vi*Vj*(G_ij*cos(θij) + B_ij*sin(θij)) → z(실제 측정)와 h(x)(상태로 예측한 값)의 차이가 작아지는 x를 찾는 게 목표

STEP 3 — 목적함수 J(x): "얼마나 안 맞나"를 한 숫자로

측정값 z와 예측값 h(x)의 차이(잔차 r)를 정밀도로 가중해서 제곱·합산한 것이 J입니다.

잔차: ri = zi - hi(x) (측정 - 예측) 가중치: wi = 1/sigma_i^2 (정밀한 계측일수록 큰 가중치) 목적함수: J(x) = Sum_i wi*ri^2 = Sum_i (zi - hi(x))^2 / sigma_i^2 예) 전압계(sigma=0.004)는 w=62,500, 전력계(sigma=0.008)는 w=15,625 → 전압계가 4배 더 정밀하니 그 오차에 4배 더 민감하게 반응

이 J(x)를 최소로 만드는 x가 "가장 그럴듯한 상태" xhat입니다.

STEP 4 — 어떻게 최소를 찾나: Gauss-Newton 반복

h(x)가 sin·cos로 비선형이라 한 번에 못 풉니다. NR과 똑같이 "현재 추정 → 보정 → 반복"합니다. 핵심은 야코비안 H(상태를 조금 바꾸면 계측 예측이 얼마나 변하나)입니다.

H = ∂h/∂x (31x13 행렬, 코드에선 수치미분 e=1e-6 로 계산) 정규방정식: (H^T*W*H) Δx = H^T*W*(z - h(x)) └──G(13x13)──┘ └── 잔차 가중합 ──┘ Δx = G^(-1) H^T*W*r ← 이번 반복의 상태 보정량 x ← x + Δx |Δx| < 1e-7 이 될 때까지 반복 (실제 4회면 수렴)
G = H^T*W*H 가 뭐냐: "정보행렬(information matrix)"입니다. 각 계측이 각 상태에 대해 가진 정보를 정밀도로 가중해 13×13로 압축한 것. 이걸 역행렬 취해 보정량을 분배합니다. 31개 계측의 정보가 13개 상태로 "녹아드는" 과정입니다.

STEP 5 — 불량을 어떻게 잡나 ① 검출: χ² 검정

추정이 끝나면 남은 J(xhat) 값 자체가 "전체적으로 측정들이 얼마나 안 맞았나"를 말해줍니다. 정상 노이즈만 있으면 J는 통계적으로 자유도=dof인 카이제곱 분포를 따릅니다.

정상이라면: J(xhat) ~ χ^2(dof=18) 분포 → 기댓값 ≈ dof = 18 근처 임계값: χ^2_0.99(18) = 34.83 (99% 신뢰: 정상이면 이 값 넘을 확률 1%) 판정: J > 34.83 이면 → "어딘가 불량 데이터 있다!" 경보 실제 실행값: 정상(노이즈만): J ≈ 17 < 34.83 → 통과 +45MW 주입: J ≈ 1900 >> 34.83 → 검출

J≈17이 자유도 18과 비슷한 게 "정상"의 증거입니다. 불량 하나가 들어가면 그 큰 잔차가 제곱(²)되고 가중(/σ²)돼서 J가 폭발적으로 커집니다.

STEP 6 — 불량을 어떻게 잡나 ② 식별: 정규화잔차(LNR)

"불량이 있다"까지는 χ²로 알았지만, 어느 계측이 범인인지는 잔차 크기만으로는 부족합니다. 잔차를 그 표준편차로 나눠 정규화해야 공정하게 비교됩니다.

잔차 공분산: Omega = R - H*G^(-1)*H^T (R=diag(sigma^2), G=H^T*W*H) → Omega_ii = 그 계측 잔차가 자연스럽게 가질 수 있는 분산 정규화잔차: rn_i = |ri| / sqrt(Omega_ii) ← "표준편차 몇 배나 튀었나" 식별 규칙: 가장 큰 rn_i 가 범인. (임계 3.0 초과 시 확정) → 결과: P7-3 (Bus7→Bus3 유효조류) 지목 = 주입한 그 계측 정확히 적발 ✓
왜 Omega로 나누나 (그냥 |ri|로 비교하면 안 되나?): 계측마다 "원래 튈 수 있는 정도"가 다릅니다. 연결이 많은 모선의 계측은 다른 계측들이 잡아줘서 잔차가 작게 나오고, 외딴 계측은 크게 나옵니다. 그래서 |ri|를 그대로 비교하면 불공정합니다. sqrt(Omega_ii)로 나눠 "각자 기준에서 몇 배 튀었나"로 환산해야 진짜 범인이 1등으로 올라옵니다.

STEP 7 — Smearing(오염 파급): 빨강 하나, 노랑 여러 개

불량은 P7-3 하나뿐인데, 화면엔 인근 계측들도 노란색(잔차 약간 큼)으로 보입니다. 이유는 STEP 4에서 추정이 그 오염된 값에 끌려가며 상태 xhat가 살짝 왜곡됐기 때문입니다.

P7-3에 +45MW 오류 → 추정이 그 값에 맞추려 Bus7 주변 V·θ를 살짝 틀리게 추정 → 그 틀린 상태로 예측한 인근 계측들도 잔차가 덩달아 커짐 (smearing) 빨간 다이아몬드(P7-3): 정규화잔차 압도적 1등 → 진짜 불량 (실제 +45MW) 노란색(Bus7 인근): 잔차 약간 큼 → 실제 오류 아님, 파급 효과일 뿐
그래서 한 번에 하나씩: 노란 것까지 다 지우면 멀쩡한 계측을 버립니다. 실무는 최대 rn 하나 제거 → 다시 추정 → 또 확인을 반복합니다. P7-3을 빼고 다시 돌리면 노란색들이 정상으로 돌아오고 J도 17 근처로 떨어집니다.
