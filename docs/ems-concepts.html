Q 01

Slack 버스와 위상각 기준점Slack Bus & Phase Angle Reference

왜 Slack 버스가 필요한가?

조류 방정식은 위상각의 차이만 관계하므로, 절대 위상을 하나 고정해야 합니다. 이 기준 모선이 Slack 버스입니다. 동시에 계통의 발전·부하·손실 불일치를 자동으로 메우는 수급 균형자 역할도 합니다.

Slack 버스의 두 역할
① 위상 기준: θslack = 0° 고정 (모든 각도는 이 기준과의 차)
② 균형자: 미지의 총 발전 P, 무효 Q를 자동 공급·흡수 (음수면 흡수)

좀 더 풀어서 — Slack을 "왜, 어떻게" 정하나

전력조류는 미지수와 방정식의 수가 딱 맞아야 풀립니다. 만약 모든 발전기의 P를 다 지정해버리면 손실(I²R)을 미리 알 수 없다는 모순이 생깁니다. 손실은 조류를 풀어야 나오는데, 발전량은 손실까지 포함해 정해야 하니까요. 이 "닭-달걀" 문제를 푸는 장치가 Slack입니다.

전체 발전 = 전체 부하 + 전체 손실 ↑ 풀기 전엔 모르는 값 → 발전기 하나(Slack)의 P를 비워두고, 나머지를 다 푼 뒤 "모자라거나 남는 만큼"을 그 발전기가 자동으로 메우게 한다 = Slack이 손실 + 불균형을 마지막에 흡수·공급
왜 Bus1을 Slack으로 골랐나 (설계 의도): Slack은 계통의 모든 불균형을 순간적으로 받아내야 하므로 가장 크고, 예비력(여유 출력)이 많고, 응답이 빠른 주력 발전기가 맡습니다. 실제 계통에서는 대형 기력·원자력·수력 같은 기저 발전기나 계통연계점(타 변전소와의 연결)이 이 역할을 합니다. 그래서 저는 두 발전기 중 더 "튼튼한 주전원" 역할을 Bus1에 부여하고 Slack으로 지정했습니다.

위상 기준이라는 두 번째 역할도 직관적으로 보면, 시계의 12시 방향을 정하는 것과 같습니다. 모든 모선의 위상각은 "Bus1보다 얼마나 앞서거나 뒤졌나"로만 의미가 있고, 그 0점 기준이 없으면 각도들이 공중에 떠버립니다(부정해). 그래서 θ₁=0°로 못 박습니다.

버스 유형알려진 값구하는 값
Slack Bus1V=1.05, θ=0°P, Q
PV Bus2P=50MW, V=1.05θ, Q
PQ Bus3,4,5P, Q (부하)V, θ
PQ Bus6,7P=0, Q=0V, θ

부하 Q = P·tan(acos(0.95)) ≈ 0.3287·P. Bus1이 Slack (가장 큰 발전·예비력 가정).

Q 02

위상각의 방향과 의미Phase Angle Direction

전류는 위상이 앞선 곳(θ 큰 곳) → 뒤진 곳(θ 작은 곳)으로 흐릅니다. Slack(Bus1)을 0°로 고정한 실제 수렴 결과:

Bus2 (θ = +0.108°) ← PV 발전기: 기준보다 앞섬 → 전력 주입 Bus1 (θ = 0.000°) ← Slack 기준 Bus6 (θ = −0.564°) ← 집전 모선 Bus7 (θ = −5.313°) ← 변압기 2차(배전) Bus4 (θ = −7.399°) ← 부하 Bus3 (θ = −7.830°) ← 부하 Bus5 (θ = −8.266°) ← 부하 최대 → 위상 최저 전력은 위상 "내리막"으로 흐른다: 발전(+0.1°) → 부하끝단(−8.3°)
변압기에서 각도가 크게 꺾이는 이유: TR의 리액턴스(x=0.10)가 선로보다 커서 Bus6(−0.56°)→Bus7(−5.31°)에서 약 4.7° 급강하합니다. 위상차가 클수록 그 설비로 많은 유효전력이 통과한다는 뜻 (TR에 전 부하 90MW 집중).
Pij ≈ (Vi·Vj / Xij)·sin(θi − θj) 각도차 ↑ → 전송 유효전력 ↑. 실계통에서 30° 이상이면 안정도 위험.
Q 03

퍼유닛(per-unit) 시스템Per-Unit System

전압등급이 다른 구간(이 계통은 154kV·22.9kV)을 실제 단위로 계산하면 변압기 권선비가 식을 복잡하게 만듭니다. pu는 각 구간 정격으로 무차원화해 변압기를 "투명"하게 만듭니다.

전압 pu (구간마다 기준 다름)

V[pu] = V[kV] / V_base[kV] // Bus1,2,6: base 154kV // Bus7,3,4,5: base 22.9kV // 1.05pu → 154×1.05 = 161.7kV

전력 pu (계통 전체 하나)

P[pu] = P[MW] / 100MVA // S_base = 100MVA 고정 // 90MW → 0.90pu // 변압기 있어도 동일 기준
⚠ 핵심: 전압 pu의 기준(V_base)은 변압기마다 바뀌지만, 전력 pu의 기준(100MVA)은 계통 전체에서 하나로 고정.
임피던스 환산: Z[pu] = Z[Ω] × S_base / V_base² 코드의 r,x는 이미 pu값으로 입력되어 별도 환산 불필요.
Q 04

임피던스·어드미턴스의 출처Where Do r, x, y Come From?

r + jx 값은 어떻게 정했나?

이 값들은 실측이 아니라 전압등급·선로특성을 현실적으로 흉내 낸 표준 pu 예시값입니다. 핵심 지표는 X/R 비율로, 송전계통은 항상 X ≫ R (도체가 굵어 저항은 작고 인덕턴스가 지배적)입니다.

선로r (pu)x (pu)X/R특성
선로1,2 (154kV 송전)0.010.033.0굵은 송전선, 저저항
변압기 TR (154/22.9kV)0.020.105.0누설리액턴스 큼 → X/R 최대
선로4,5,6 (22.9kV 배전)0.040.153.75가늘고 긴 배전선 → 임피던스 큼

어드미턴스 = r+jx 통째로의 복소수 역수 ✓

y = 1/(r+jx) = (r − jx)/(r²+x²) = g + jb 선로1: 1/(0.01+j0.03) = (0.01−j0.03)/0.001 = 10.00 − j30.00 TR : 1/(0.02+j0.10) = (0.02−j0.10)/0.0104 = 1.92 − j9.62 선로4: 1/(0.04+j0.15) = (0.04−j0.15)/0.0241 = 1.66 − j6.22

네, "통째로를 각각 역수"가 맞습니다. 단 복소수 역수이므로 분모를 유리화(r²+x²로 나눔)합니다.

Q 05

Ybus 행렬 — KCL 유도와 부호 규칙Bus Admittance Matrix

대각항 = "연결된 모든 선로 어드미턴스의 합"

Y11 = 10−j30 (Bus1엔 선로1 하나만) Y66 = (10−j30)+(10−j30)+(1.92−j9.62) = 21.92 − j69.62 (선로1+2+TR) Y77 = (1.92−j9.62)+3×(1.66−j6.22) = 6.90 − j28.29 (TR+선로4+5+6)

비대각항 Y₁₆ = −10+j30 은 왜 "부호 반대"인가?

정확한 규칙은 "두 모선 사이 선로 어드미턴스의 부호 반전"입니다:

Y16 = −(1·6 사이 선로 어드미턴스) = −(10−j30) = −10 + j30
오해 주의: Y16이 마치 −Y11처럼 보이는 건 우연입니다. 모선1은 선로가 딱 하나라 Y11 = 그 선로값이 되어 그렇게 보일 뿐. 모선6은 다릅니다 — Y66 = 21.92−j69.62 지만 Y16 = −10+j30 ≠ −Y66. 항상 선로 어드미턴스 기준입니다.

왜 부호를 반대로? — KCL(전류법칙) 유도

모선 i 주입전류: I_i = Σⱼ yᵢⱼ·(V_i − V_j) ← 이웃 j로 흐르는 전류 합 = (Σⱼ yᵢⱼ)·V_i − Σⱼ yᵢⱼ·V_j └ V_i 계수 = +합 (대각 Yᵢᵢ) └ V_j 계수 = −yᵢⱼ (비대각 Yᵢⱼ)

전류가 전압 차(V_i − V_j)에 비례하기 때문에 내 전압엔 (+), 이웃 전압엔 (−)가 붙습니다. "이웃 전압이 높으면 전류가 나에게 역류한다"의 수학적 표현입니다.

0 = 직접 연결 안 됨 ✓

Y13 = 0 → 모선1·3은 직접 선로 없음 (6,7을 거쳐 연결). 0의 위치만 봐도 결선도를 복원할 수 있습니다.

Q 06

N-1: 선로 개방 시 Ybus 변화Contingency on the Ybus

계통해석이란 곧 "개방된 상태의 Ybus를 다시 짜서 조류해석을 푸는 것"입니다. 세 대표 케이스를 실제 엔진으로 확인했습니다.

❓ 선로1을 열면 전 계통 정전인데, 선로2를 열면 왜 정전이 아닌가?

핵심은 연결성 탐색(BFS)이 항상 Slack(Bus1)에서 출발한다는 점입니다.

🔴 선로1 개방 (Bus1↔Bus6)

Bus1(Slack) ─╳─ Bus6 ── Bus7 ── 부하 Bus2(PV) ──────┘

Bus1(Slack)에서 BFS 시작 → 선로1이 유일한 통로인데 끊김 → 도달 가능: {Bus1}만 → Bus2,6,7,3,4,5 전부 고립 → 90MW 전 계통 정전

✅ 선로2 개방 (Bus2↔Bus6)

Bus1(Slack) ──── Bus6 ── Bus7 ── 부하 Bus2(PV) ─╳─────┘

Bus1(Slack)에서 BFS 시작 → 선로1은 살아있음 → Bus6·7·3·4·5 도달 → Bus2만 고립 (발전기뿐, 부하 없음) → 부하 정전 0MW

왜 Bus2 고립은 괜찮은가?
Bus2는 발전기 전용 모선으로 소비 부하가 없습니다. Bus2가 탈락하면 50MW 발전이 사라지지만, Slack(Bus1)이 균형자 역할로 그 50MW를 자동 보충합니다. 수용가 입장에서는 전력 공급이 끊기지 않습니다.

비유: 발전소 2개(Bus1·Bus2) 중 하나(Bus2)가 꺼졌을 때, 나머지 발전소(Bus1)가 출력을 올려 커버하는 것과 같습니다.
개방 선로BFS 도달 가능 모선고립 모선정전 부하이유
선로1 (Slack 연계) Bus1만 Bus2,6,7,3,4,5 90 MW Slack이 섬이 됨 → 기준전원 상실
선로2 (PV 연계) Bus1,6,7,3,4,5 Bus2만 0 MW 발전기만 탈락, Slack이 보충
결론: 선로1이 특별히 위험한 이유는 Slack이 계통에 연결되는 유일한 통로이기 때문입니다. Slack을 잃으면 나머지가 물리적으로 연결돼 있어도 해석 기준 자체가 사라집니다. 이것이 "Slack 연계선은 반드시 2회선 이상"을 요구하는 이유입니다.

선로1 개방 Ybus

y₁₆이 빠지면 모선1은 연결선이 없어 Ybus가 사실상 비어버립니다.

===== 선로1 개방 ===== 가압모선: 1 | 고립: 2,6,7,3,4,5 Ybus = [ 0 ] ← 모선1 혼자, 연결선 없음 → 90MW 전 부하 정전 (기준전원 상실 = 블랙아웃)
참고: 모선2(PV)+6+7+부하는 물리적으로 서로 연결돼 있지만, Slack(기준)을 잃은 "섬"이라 이 모델은 정전 처리합니다. 실계통이라면 모선2가 새 기준이 되어 섬 운전(islanding)을 시도하나, 발전 50MW < 부하 90MW라 결국 부하 차단이 불가피합니다.

선로4 개방 → 모선3만 고립 (30MW)

y₇₃(1.66−j6.22)이 빠지고, 모선3은 선로4 하나로만 연결돼 고립됩니다. 모선7 대각항이 그만큼 감소:

===== 선로4 개방 ===== 가압모선: 1,2,6,7,4,5 | 고립: 3 Y77: 6.90−j28.29 → 5.24 − j22.06 (y₇₃ 만큼 감소) ✓ → 모선3의 30MW만 상실, 나머지 계통은 정상 수렴
주의: 개방되는 선로4는 모선7↔모선3이므로 빠지는 성분은 Y₃₃·Y₇₃·Y₃₇ 와 Y₇₇ 감소분입니다. 모선4는 선로5로 연결돼 그대로 살아있습니다.

선로4 개방 시 실제 Ybus 행렬 (7×7 → 6×6)

모선3이 고립되면 행렬에서 3번 행·열이 통째로 삭제됩니다. 남는 가압모선은 {1,2,6,7,4,5} 6개. 아래는 엔진이 실제로 만든 행렬(G + jB)입니다. 평상시 7×7과 비교해 3번 줄이 사라지고 Y₇₇이 6.90−j28.29 → 5.24−j22.06으로 줄어든 것만 다릅니다.

이 행렬로 PF를 어떻게 푸나? — 4단계
  1. 고립 판정: Slack(Bus1)에서 BFS로 도달 가능한 모선만 추림 → Bus3은 도달 불가 → 해석 대상에서 제외(정전 처리, 30MW 상실 확정).

Q 07

Newton-Raphson 반복 추적 — 손 계산으로 이해NR Iteration Trace — Step by Step

먼저: "mismatch(불일치)"란 무엇인가?

NR은 "우리가 원하는 전력(P_spec, Q_spec)"과 "현재 추정한 전압으로 계산한 전력(P_calc, Q_calc)"의 차이를 줄여나가는 과정입니다.

mismatch = P_spec − P_calc(V, θ) Q_spec − Q_calc(V, θ) 이 값이 0에 가까울수록 → 전압 추정값이 실제에 가깝다 = 수렴
직관적 비유: 내가 "버스6에 전력이 0MW 들어오길 원한다"고 지정했는데, 현재 전압 추정값으로 계산해보니 −300MW가 흐른다고 나오면 mismatch = 300MW. 이 차이를 없애도록 전압을 조정하는 것이 NR입니다.

STEP 0 — Flat Start (초기 추정값 설정)

아무것도 모르는 상태에서 시작. 모든 전압을 1.0pu, 모든 위상각을 0°으로 설정합니다. 단, 발전기(Bus1·2)는 설정값(1.05pu)으로 시작합니다.

초기값: V₁=1.05, V₂=1.05 ← 발전기는 설정값 V₆=V₇=V₃=V₄=V₅=1.00 ← 나머지는 1.0으로 가정 θ₁=θ₂=θ₆=θ₇=θ₃=θ₄=θ₅ = 0° ← 모든 위상각 0으로 가정

STEP 0 — mismatch 계산 (왜 3.0pu인가?)

위상각이 전부 0이면 cos(θᵢ−θⱼ)=1, sin(θᵢ−θⱼ)=0이 되어 Q 계산식이 단순해집니다:

Q_calc_i = −Vᵢ × Σⱼ(Vⱼ × Bᵢⱼ) (모든 θ=0일 때 sin=0이라 G항 사라짐) Bus6에 대해 계산: B₆₁=+30, B₆₂=+30, B₆₆=−69.62, B₆₇=+9.62 (Ybus B행렬에서) Σ = V₁×30 + V₂×30 + V₆×(−69.62) + V₇×9.62 = 1.05×30 + 1.05×30 + 1.00×(−69.62) + 1.00×9.62 = 31.50 + 31.50 − 69.62 + 9.62 = 3.00 Q_calc_6 = −1.00 × 3.00 = −3.00 pu Q_spec_6 = 0 (Bus6은 부하·발전 없는 집전모선 → Q = 0 이어야 함) ΔQ₆ = Q_spec − Q_calc = 0 − (−3.00) = +3.00 pu ← max mismatch!
왜 Bus6의 ΔQ가 가장 클까?
Bus6은 발전기(1.05pu)와 나머지 계통(1.0pu) 사이에 끼어있는 집전모선입니다. 서셉턴스가 큰 두 선로(B=30)가 양쪽에서 잡아당기는데, 전압 단차(1.05−1.00=0.05pu)가 B=30이라는 큰 값과 곱해져 3.0pu라는 큰 불일치가 생긴 것입니다. 선로 임피던스가 작을수록(=B가 클수록) 이 효과가 커집니다.

STEP 1 — Jacobian으로 보정량 계산

mismatch를 없애려면 V와 θ를 얼마나 바꿔야 하는지 계산합니다. 이때 사용하는 것이 야코비안(J)입니다.

J × Δx = mismatch J = [∂P/∂θ ∂P/∂V] ← 전압/각도를 조금 바꿀 때 P가 얼마나 변하는지 [∂Q/∂θ ∂Q/∂V] ← 전압/각도를 조금 바꿀 때 Q가 얼마나 변하는지 이 식을 Gauss 소거법으로 풀어 Δθ, ΔV를 구함 → 전압/각도 갱신: θ ← θ + Δθ, V ← V + ΔV

각 반복의 mismatch (실제 엔진 실행값)

iter 0
3.000e+0 pu = 300 MW  (ΔQ at Bus6)
iter 1
1.759e−1 pu = 17.6 MW  (ΔQ at Bus6)
iter 2

이 숫자들은 도대체 어디서 나오나? — 한 반복의 내부 동작

위 표의 각 줄은 마법의 숫자가 아니라, 매 반복마다 엔진이 똑같은 4단계를 반복해서 나온 값입니다. powerflow.ts의 for-루프가 정확히 이 일을 합니다.

[루프 1바퀴 = 1 iteration] ① 현재 V, θ로 모든 모선의 실제 주입전력을 계산 Pc[i], Qc[i] = busInjection(V, θ, i) ← Ybus를 써서 계산 ② 원하는 값 − 계산된 값 = mismatch 벡터를 만든다 ΔP[i] = P_spec[i] − Pc[i] (Slack 제외 모든 모선) ΔQ[i] = Q_spec[i] − Qc[i] (PQ 모선만) ③ 이 벡터에서 절댓값이 가장 큰 성분 = 그 줄의 "maxMismatch" maxMismatch = max( |ΔP|, |ΔQ| 전부 중 최댓값 ) ④ J·Δx = mismatch 를 풀어 ΔV, Δθ 를 구하고 V,θ 갱신 → 다음 바퀴에서 ①의 Pc, Qc가 달라짐 → mismatch가 줄어듦
왜 iter0·1은 "ΔQ at Bus6", iter2부터는 "ΔP at Bus7"로 바뀌나?
maxMismatch는 매 반복마다 "지금 가장 안 맞는 곳"을 가리킵니다.
  • iter0·1: flat start 직후엔 발전기(1.05pu)와 집전모선(1.0pu)의 전압 단차 때문에 Bus6의 무효전력(Q)이 가장 안 맞음 → ΔQ₆가 최대 (3.00 → 0.176).
  • iter2 이후: 전압 크기는 거의 자리잡혔고, 이제 위상각(θ)이 결정하는 유효전력(P)의 미세한 오차가 남음. 부하가 먼 Bus7쪽 ΔP가 마지막까지 남아 최대가 됨 (0.0015 → 8.6e−7).
즉 "어느 모선이 최대냐"가 바뀌는 것은 버그가 아니라, 전압(Q) 먼저 잡고 → 위상각(P) 나중에 다듬는 NR의 자연스러운 수렴 순서입니다.

검산: iter0의 3.000pu는 바로 위 STEP 0에서 손으로 계산한 ΔQ₆=+3.00pu와 정확히 일치합니다. iter1부터는 갱신된 V·θ로 ①을 다시 돌려 나온 값이라 손 계산이 번거로워 엔진 출력을 그대로 실었습니다.

왜 이렇게 빠르게 줄어드나? — 이차수렴

뉴턴법은 오차가 매 반복마다 제곱으로 줄어드는 특성이 있습니다. 이를 이차수렴(quadratic convergence)이라 합니다.

오차가 e이면, 다음 반복 후 오차 ≈ k × e² iter0 → iter1: 3.000 → 0.176 (58배 감소) iter1 → iter2: 0.176 → 0.00152 ≈ 0.176² × 0.049 (116배) iter2 → iter3: 0.00152 → 8.6e−7 ≈ 0.00152² (1770배) iter3 → iter4: 8.6e−7 → 2.0e−13 ≈ (8.6e−7)² (제곱 그대로!) 유효 자릿수: 1자리 → 1자리 → 3자리 → 6자리 → 13자리 ↑ 2배씩 늘어남 = 이차수렴
이차수렴이 실용적인 이유: 일반 반복법(선형수렴)은 100번 반복해도 2자리 개선이지만, NR은 4번으로 13자리 개선됩니다. 대형 계통(1000개 버스 이상)도 보통 5~10회 안에 수렴하는 이유입니다.

Q 07-C

시뮬레이터 한계: 섬 운전과 현실의 차이Simulation Limit vs Real Islanding

실험: 부하=30MW, PV=30MW, 선로1 개방

발전과 부하가 완벽히 균형을 이루는 상태에서 Slack 연계선(선로1)을 개방하면 어떻게 될까요?

시뮬레이터 결과

가압모선: [Bus1]만 고립모선: [Bus2,6,7,3,4,5] 총발전: 0 MW 총부하: 0 MW → "전 계통 정전" 판정

PV=30MW, 부하=30MW로 완벽히 균형인데도 정전으로 처리.

실제 계통

  • Bus2(PV)+Bus6+Bus7+Bus3~5 섬이 물리적으로 살아있음
  • 발전(30MW) = 부하(30MW) → 주파수 60Hz 유지
  • Bus2의 AVR이 전압 유지
  • 섬 운전(Islanding) 성공 가능

왜 시뮬레이터가 틀린 판단을 하나?

Newton-Raphson은 Slack 버스가 반드시 1개 있어야 수학적으로 풀립니다. Bus2는 PV 버스(P,V 고정)이지 Slack(θ=0 기준, 전력균형)이 아닙니다.

섬 안에 Slack 없음 → Jacobian이 특이행렬(singular, 역행렬 없음) → 연립방정식을 풀 수 없음 → "정전"으로 처리 (보수적·단순화) 실제 해결: 섬 안의 버스 하나를 임시 Slack으로 재지정하고 다시 풀어야 함 → 이 시뮬레이터는 그 처리를 구현하지 않음

발전 ≠ 부하일 때 실제 계통의 반응

상황시뮬레이터실제 계통
PV 30MW = 부하 30MW, 선로1 개방 전 계통 정전 섬 운전 성공 (균형 유지)
PV 50MW > 부하 30MW 전 계통 정전 주파수 상승 → OFR(과주파수계전기) → PV 출력 감발 → 안정화
PV 10MW < 부하 30MW 전 계통 정전 주파수 하락 → UFR(저주파수계전기) → 부하 차단 → 일부 생존
실무 적용: 실제 EMS에는 고립 감지(Islanding Detection)와 섬 운전 제어(Island Operation) 기능이 별도로 존재합니다. 재난 시 특정 구역을 의도적으로 분리해 자립 운전시키는 계획 섬 운전(Intentional Islanding)도 활용합니다. 이 시뮬레이터는 그 복잡함을 생략하고 "Slack 없으면 정전"으로 단순화한 교육용 모델입니다.
Q 07-D

임피던스는 고정인데 부하·발전을 바꿔도 되나? & PF는 암산과 무엇이 다른가Fixed Impedance vs Variable Injection · Why Power Flow ≠ Mental Math

질문 1 — PV·부하를 바꾸면 R, X도 바뀌어야 하는 것 아닌가?

아니요. 바뀌지 않습니다. 여기에 전력조류의 핵심 개념이 있습니다. 계통 방정식은 두 종류의 양으로 나뉩니다.

구분무엇무엇이 정하나부하·발전 바꾸면?
네트워크 (Ybus)선로 R, X → G, B전선의 굵기·길이·재질 (물리적 설비)안 바뀜 (고정)
주입 (Injection)P_spec, Q_spec발전기 출력·부하 소비 (운전 조건)바뀜 (당신이 조절)
핵심 직관: R·X는 "수도관의 굵기"입니다. 물(전력)을 많이 흘리든 적게 흘리든 관 자체는 그대로죠. 부하·PV를 바꾸는 것은 "양 끝에서 물을 얼마나 넣고 빼느냐"를 바꾸는 것일 뿐, 관(임피던스)을 바꾸는 게 아닙니다.
전력조류 방정식: Y · V = I(V) ← Y(Ybus)는 고정 ─┬─ ─┬── 네트워크 주입조건(P,Q_spec)이 만드는 전류 부하 30→50MW로 변경 → P_spec[bus]만 바뀜 → 같은 Y로 다시 NR → 새 V, θ ↑ Ybus는 단 한 번도 안 건드림

그래서 시뮬레이터에서 슬라이더를 움직일 때마다 엔진은 똑같은 Ybus를 재사용하고 pSched/qSched(주입 스케줄)만 갱신해서 NR을 다시 돌립니다. 임피던스가 부하에 따라 변한다면 그건 비선형 소자(전압의존부하 등)를 따로 모델링할 때의 이야기이고, 이 표준 모델에서는 R·X 상수가 맞습니다.

질문 2 — "발전량·부하량 암산"과 Ybus 기반 PF는 무엇이 다른가?

맞습니다. "누가 켜져 있고 대략 어디로 흐르나"는 암산으로 충분합니다. PF가 추가로 주는 것은 암산이 절대 줄 수 없는 정량적 정밀값입니다.

암산으로 알 수 있는 것

  • 어느 모선이 가압/정전인지 (연결성)
  • 대략의 조류 방향 (발전→부하)
  • 발전 총량 ≈ 부하 총량 + 손실

PF만이 주는 것 (암산 불가)

  • 각 모선 전압 크기·위상 (0.947pu, −8.27° 같은 값)
  • 선로별 정확한 MW·Mvar·손실
  • 무효전력(Q) 흐름 — 직관으로 거의 불가능
  • 과부하·저전압 위반 여부를 수치로

왜 암산이 안 되냐면, 전력조류는 비선형 연립방정식이기 때문입니다.

P, Q = f(V, θ) ← V와 θ가 sin·cos로 얽힌 비선형식 선로 손실 I²R 은 전류의 제곱 → 부하 2배면 손실 4배 예) "부하 90MW니까 발전 90MW" (암산) 실제 PF: 발전 = 90MW(부하) + 3.83MW(손실) = 93.83MW 게다가 Bus5 전압이 0.947pu까지 떨어짐 ← 암산으로는 못 봄
한 줄 요약: 암산은 "흐르냐/안 흐르냐"(정성적)를 보고, PF는 "얼마나, 몇 볼트로, 손실은, 위반은"(정량적)을 봅니다. 765kV 수백 모선 계통에서 0.05pu 전압강하나 95% 선로 부하율 같은 위험은 오직 PF로만 잡아냅니다. 그래서 EMS가 실시간으로 PF를 돌리는 것입니다.
Q 08

PV 버스와 AVRPV Bus & Automatic Voltage Regulator

PV 버스는 유효출력 P와 전압 V가 고정된 발전기 모선. 무효 Q와 위상 θ는 조류해석이 자동 결정합니다.

좀 더 풀어서 — Slack과 PV는 뭐가 다른가

둘 다 발전기지만 "무엇을 지정하고 무엇을 계산하느냐"가 정반대입니다. 이 차이를 이해하는 게 조류해석의 핵심입니다.

Slack (Bus1)PV (Bus2)
내가 지정V, θ(=0)P, V
해석이 계산P, Qθ, Q
P 출력자동 (불균형·손실 흡수)내가 고정 (예: 50MW)
역할균형자 + 위상기준지정량 발전 + 전압유지
왜 PV는 P를 고정할 수 있나? 실제 발전기는 운전원이 "이 발전기 60MW로 돌려"라고 출력 지령(거버너 설정)을 내립니다. 그 지령대로 P가 고정됩니다. 단 한 대(Slack)만은 손실까지 메우느라 P를 비워두는 것이고, 나머지 발전기는 전부 PV로 P를 지정합니다. 그래서 발전기 N대 = Slack 1대 + PV (N−1)대가 일반적인 구성입니다. 이 시뮬레이터는 발전기 2대라 Slack 1 + PV 1로 가장 단순한 형태를 보여줍니다.

"왜 Q는 고정 안 하나?" 발전기에는 AVR(자동전압조정기)이 달려 있어서, 전압을 1.05pu로 유지하기 위해 무효전력 Q를 실시간으로 자동 조절합니다. 즉 Q는 운전원이 정하는 게 아니라 "전압을 맞추다 보니 결정되는" 종속 변수입니다. 그래서 PV 버스는 V를 고정하고 Q를 계산값으로 둡니다. 이 계통에서 Bus2는 V=1.05pu를 지키느라 Q=+22.02Mvar을 자동 공급하는 것으로 풀립니다.

지정값

  • P = 50 MW
  • V = 1.05 pu

해석이 결정

  • θ = +0.108°
  • Q = +22.02 Mvar

AVR이 Q를 조절하는 원리

V_단자 < V_set → 계자전류 I_f ↑ → E_q ↑ → Q 공급 ↑ → V ↑ V_단자 > V_set → 계자전류 I_f ↓ → E_q ↓ → Q 흡수 ↑ → V ↓ E_q: 발전기 내부 기전력 (계자전류에 비례)
비유: AVR은 자동 수도꼭지. 전압(수압)이 낮아지면 꼭지(계자전류)를 열어 무효전력(물)을 더 보내 압력을 회복합니다. PV 모선에서 V가 고정이라는 건 "이 꼭지가 항상 작동 중"임을 가정.

실제 발전기는 Q 한계(권선 가열)가 있어 Qmax 초과 시 PV→PQ로 전환되어 전압이 떨어집니다. 이 시뮬레이터는 한계 없는 이상 모델.

Q 11-A

PF · N-1 · 상태추정 · Topology — 관계와 EMS 실행 순서How PF, N-1, SE & Topology Fit Together

이 시뮬레이터에서 하는 게 PF입니까?

네, 맞습니다. 슬라이더로 PV 발전량·부하량을 바꾸거나 차단기를 열고 닫을 때마다 내부에서 Newton-Raphson이 돌아가고, 화면에 나오는 전압·선로 부하율·과부하 경보가 모두 Power Flow(조류계산) 결과물입니다.

입력: 발전량(P, V 지정) + 부하량(P, Q 지정) + 계통 구성(어떤 차단기가 닫혀있나) ↓ Newton-Raphson 출력: 각 모선의 전압·위상각 + 각 선로의 조류·손실·부하율

PF · N-1 · 상태추정은 별개인가, 연결되나?

도구로서는 별개이지만, 실제 EMS에서는 순서대로 쌓아서 씁니다. 아래가 실제 EMS의 실시간 운영 흐름입니다.

① Topology Processing → "지금 어떤 모선이 어떻게 연결돼 있나" 파악 ↓ ② 상태추정 (WLS SE) → "현장 계측값으로 현재 V·θ 확정" ↓ ③ PF → "확정된 상태에서 조류 계산" ↓ ④ N-1 해석 → "선로 하나씩 끊어가며 PF 반복"
개념한 줄 정의방향이 시뮬레이터에서
Topology
Processing		 모선·선로 배치 + 차단기 상태로 현재 전기적 연결 구조 파악 설비 정보 → Ybus 차단기 ON/OFF 시 BFS로 연결 재파악 → Ybus 재구성
PF 운전 조건을 주면 전압·조류를 예측 조건 → 결과 (순방향) 슬라이더·차단기 조작 시 자동 실행
N-1 해석 설비 하나씩 끊어가며 PF를 반복 PF × 선로 수 차단기 하나씩 수동으로 열어보는 것이 수동 N-1
상태추정 잡음 섞인 계측값으로 현재 상태를 역추정 계측 → 상태 (역방향) 별도 탭, PF와 독립 실행

각 개념을 좀 더 풀어서

① Topology Processing

실제 변전소에는 차단기·단로기가 수십 개씩 달려 있어서, 어떤 게 닫혔냐에 따라 전기적 연결이 실시간으로 바뀝니다. 이 차단기 상태를 읽어서 "지금 어떤 모선이 어떻게 이어져 있나"를 자동으로 파악하는 것이 Topology Processing입니다. 이 시뮬레이터도 차단기를 열고 닫을 때 BFS(너비우선탐색)로 Slack에서 도달 가능한 모선을 골라내고 Ybus를 재구성합니다. 이 시뮬레이터의 계통 그림 자체(모선 7개, 선로 6개 배치)가 네트워크 토폴로지입니다.

② 상태추정 — PF보다 먼저 실행하는 이유

PF는 "발전·부하 조건을 주면 전압·조류를 계산"합니다. 그런데 실제 EMS에서는 발전·부하 조건 자체를 현장 계측기로 읽어옵니다. 그 계측값에는 잡음·오차·고장 미터가 섞여 있어서, 이걸 그대로 PF에 넣으면 틀린 결과가 나옵니다. 그래서 PF 이전에 WLS 상태추정으로 계측값을 통계적으로 정제해서 신뢰할 수 있는 V·θ 스냅샷을 만든 다음 PF를 돌립니다.

상태추정 없이: 잡음 섞인 계측값 → PF → 부정확한 조류 결과 상태추정 있을 때: 잡음 섞인 계측값 → WLS → 정제된 V·θ → PF → 신뢰할 수 있는 결과

③ PF와 N-1의 관계 — N-1은 PF의 반복 응용

N-1 해석은 별도 알고리즘이 아닙니다. PF를 선로 수만큼 자동으로 반복 실행하는 것입니다.

선로1 차단 → PF 실행 → 위반 확인 선로2 차단 → PF 실행 → 위반 확인 선로3(TR) 차단 → PF 실행 → 위반 확인 ... 선로6 차단 → PF 실행 → 위반 확인 ↓ "N-1 위반 목록" 출력 (과부하·저전압 발생 케이스)

이 시뮬레이터에서 차단기를 하나씩 수동으로 열어보는 것이 바로 수동 N-1 해석입니다. 실제 EMS는 선로가 수백 개라 이걸 자동으로 돌려 "어느 N-1에서 위반 발생" 목록을 수 초 안에 냅니다.

한 줄 요약: Topology가 지도를 그리고 → 상태추정이 현재 위치를 확인하고 → PF가 경로를 계산하고 → N-1이 "각 도로가 막히면?" 을 시뮬레이션합니다. 네 가지는 순서가 있는 한 파이프라인이며, 이 시뮬레이터는 그 파이프라인 전체를 하나의 화면에서 교육용으로 단순화해 보여줍니다.

  • 단위 MW
  • 위상각 차로 전송

  • 에너지 왕복 (L↔C)

왜 Q가 전압의 주역인가?

ΔV ≈ (P·R + Q·X) / V X ≫ R 이므로: ΔV ≈ Q·X / V ← Q가 지배! P를 2배 → ΔV 거의 불변, Q를 2배 → ΔV 2배 (전압 급락)
물 배관 비유: 유효전력 P = 유량(흐르는 물의 양), 무효전력 Q = 수압. 수압(Q)이 낮으면 아무리 많은 물(P)도 끝단 도달 전 압력이 떨어짐 → Q 부족 = 전압 붕괴 위험.
